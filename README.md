# Agile Planning Project (DevOps Student Lab)

This repository contains an Agile planning project completed as part of the IBM Skills Network course on Agile and Scrum.

It demonstrates how modern development teams manage work using Agile methodology, Scrum practices, and Kanban boards.

---

## 📌 Project Description

The goal of this project is to simulate a real-world Agile workflow using GitHub and ZenHub.

It covers the full lifecycle of Agile planning:
- Defining user requirements  
- Creating and managing a product backlog  
- Prioritizing tasks  
- Planning a sprint  
- Tracking progress using a Kanban board  

---

## 🧠 Agile Concepts Applied

This project demonstrates practical understanding of:

- Agile Development  
- Scrum Framework  
- User Stories  
- Product Backlog  
- Sprint Planning  
- Kanban Workflow  
- Continuous Improvement  

---

## 🧾 Example User Stories

Below are examples of user stories used in this project:

- As a user, I need a product catalog so that I can browse available items  
- As a user, I want to add items to a cart so that I can purchase them  
- As an admin, I want to manage inventory so that stock is always updated  

Each story follows Agile best practices and is tracked using GitHub Issues.

---

## 📋 Backlog Management

The project includes:
- Prioritized backlog  
- Clearly defined user stories  
- Issue labels for categorization  
- Story point estimation (if applied)  

Backlog refinement ensures continuous improvement and clarity of tasks.

---

## 🗂️ Kanban Workflow

The workflow is managed using a Kanban board with the following stages:

- Icebox → Ideas and future work  
- Backlog → Ready for development  
- In Progress → Currently being worked on  
- Done → Completed tasks
